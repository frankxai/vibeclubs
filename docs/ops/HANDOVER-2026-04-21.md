**Architecture — the one decision that matters.** `ADR-002-FORMAT-NOT-PLATFORM.md`

existing platforms; it is NOT a WebRTC product. If anyone proposes "let's add

and `HANDOVER-SUPERSEDED.md`.

**Voice system — ADR-level.** `ADR-003-VOICE-SYSTEM.md` pins the five-word
vocabulary (vibeclub / host / crew / lock in / ship), forbidden list
(community / members / rooms / platform / engage / unlock / AI-powered /
Cloud / capital-T feature names like Stamps / Anchors / Field / Witness),
canonical hero ("Claude Code + your crew + a soundtrack. Host a vibeclub.").
Run `/voice-audit` before any copy-touching merge.

**Env var soft-fallback is intentional.** `/explore` and `/club/[slug]` return
empty data when `NEXT_PUBLIC_SUPABASE_URL` is unset so design work can proceed
dry. Do not "fix" this by throwing.

**`@vibeclubs/ai-witness` name is permanent.** UI calls it "recap." Package
name is technical shorthand for the witness pattern. Devs read code, not
marketing. Do not rename.

**Packages use `"main": "./src/index.ts"`, not compiled dist.** `next.config.mjs`
lists them in `transpilePackages` — zero-build inner loop. Keep it that way.

**AudioContext needs a user gesture.** Extension boots the mixer on first
click. Not a bug.

**Apostrophe encoding.** Playbook data uses real apostrophes in template strings.
If `&apos;` leaks into rendered HTML, it's a regression.

## Critical Context

**Architecture — the one decision that matters.** `ADR-002-FORMAT-NOT-PLATFORM.md`
supersedes ADR-001. Vibeclubs is a **format**, not a platform: a Chrome extension
overlays existing platforms (Meet/Discord/Zoom), it is not a WebRTC product.
`apps/web/app/r/[room]/page.tsx` is a legacy LiveKit skeleton marked do-not-extend.
If anyone suggests "let's add LiveKit rooms" they're re-litigating a dead
architecture — point them at ADR-002 and `HANDOVER-SUPERSEDED.md`.

**Voice system — ADR-level, not style-guide-level.** `ADR-003-VOICE-SYSTEM.md`
pins the five-word vocabulary (vibeclub, host, crew, lock in, ship), forbidden
list (community, members, rooms, platform, engage, unlock, AI-powered, Cloud,
and capital-T feature names like Stamps/Anchors/Field/Witness), and the
canonical hero sentence ("Claude Code + your crew + a soundtrack. Host a
vibeclub."). A prior agent proposed an ornate taxonomy — rejected. Do not
reintroduce capital-noun features. Run `/voice-audit` before shipping.

**Env var soft-fallback is intentional.** `/explore` and `/club/[slug]` return
empty data when `NEXT_PUBLIC_SUPABASE_URL` is unset so the site still renders
for design work. Don't "fix" this by throwing.

**`@vibeclubs/ai-witness` stays named as-is.** UI surfaces call it "recap."
Package name is technical shorthand for the witness pattern — devs read code,
not marketing. Do not rename the package.

**Packages use `"main": "./src/index.ts"`, not compiled dist/.** `next.config.mjs`
lists them in `transpilePackages`. Zero-build inner loop. Do not add a build
step to inner consumers.

**AudioContext needs a user gesture.** The extension overlay boots the mixer
on first click — don't fix "why doesn't audio play before click" as a bug.
It's a browser rule.

**Apostrophe encoding:** the playbook data (`apps/web/app/playbook/playbook-data.ts`)
was rewritten on Apr 21 to use real apostrophes in template strings. If you
ever see `&apos;` leaking into rendered HTML, it's a regression — template
strings use plain `'`, JSX escapes are only for JSX attributes.
